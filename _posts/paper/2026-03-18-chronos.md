# 摘要
简述 Chronos 提出在长期对话记忆中引入结构化事件日历与非结构化回顾日历，解决时间性推理与跨轮次检索的问题。

# 主要贡献
- 双日历结构：事件日历（⟨主体-谓语-对象⟩ + 时间窗）与 turns 日历（原始对话转）
- 动态提示（dynamic prompting）为每个查询生成检索指导
- 基于图文嵌入的初步检索 + rerank 的混合检索策略
- 通过工具调用循环实现对日历的多跳时间推理

# 方法概览
- 事件提取管线：将对话转化为带时间范围的事件元组，生成别名以提升检索召回
- 动态提示：为每个问题生成检索清单（目标实体/属性、时间约束、推理策略）
- 初始检索： turns 日历进行 dense 检索+ rerank，扩展上下文
- Chronos Agent：在日历上进行向量检索与精确检索的混合调用（grep + vector）

# 实验设置
- 评估数据集：LongMemEvalS，共 500 题，6 类任务
- 模型基线：GPT-4o 与 Claude Opus 4.6 两个配置（Low/High）
- 检索/嵌入/排序：text-embedding-3-large，Cohere Rerank v3
- 指标：总体准确率，类别准确率

# 关键结果
- Chronos Low: 92.60%
- Chronos High: 95.60%
- 主要贡献来自事件日历与增强的检索/提示策略

# 讨论与局限
- 双日历增加存储与推理成本
- 部分类别（如 SSP）对某些模型存在挑战
- 推理时间相对较长，需对在线任务权衡

# 参考要点
- Chronos 的核心思想是将临时性事实和长期对话的记忆分离并以日历形式组织，结合动态检索与多跳推理来回答复杂的时间相关查询。
